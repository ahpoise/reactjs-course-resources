# Conditional rendering & Review - Sessão 5

## O que muda em relação à Sessão 4

1. **Empty state quando o filtro não devolve utilizadores.** Criamos `src/components/EmptyState.jsx` (recebe `message` como _prop_) e usamos um _ternary operator_ no `<App>` para renderizar `<EmptyState>` quando `filtered.length === 0` ou a lista habitual caso contrário. A mensagem distingue _filtro vazio_ (`selectedTag` tem valor) de _lista global vazia_.
2. **Painel de detalhe condicional ao lado da lista.** Criamos `src/components/UserDetail.jsx` que mostra `name`, `age`, `avatar` e _tags_ (com _guard_ duplo `user.tags && user.tags.length > 0` para a secção das _tags_). No `<App>`, _ternary operator_: `<UserDetail>` se há `selectedUser`, senão um `<EmptyState>` com mensagem "Seleciona um utilizador para ver detalhes.".
3. **Guard `&&` no `<select>` de filtro.** Envolvemos `<label>` + `<select>` em `{users.length > 0 && (<>...</>)}`. Se não houver utilizadores na lista global, o filtro desaparece. O `> 0` é essencial: `users.length` sozinho avaliaria para `0` e JSX renderizava esse `0` literal no DOM.
4. **Botão "Limpar seleção" no `<UserDetail>`.** O componente passa a aceitar uma _prop_ `onClear` e renderiza um botão que a chama. No `<App>`, `onClear={() => setSelectedUser(null)}` (uma _arrow function_, não a referência direta da _setter_, para evitar que o `onClick` chame `setSelectedUser` com o `SyntheticEvent`).
5. **_Counter_ contextual "X de Y" por cima da lista.** Adicionamos um `<p>` no topo do `<Section title="Diretório">` com texto condicional via _ternary operator_: quando `filtered.length < users.length`, mostra "Mostrando X de Y utilizadores."; caso contrário, "Y utilizadores.". Os dois números são derivações, calculados no _render_, sem `useState`.
6. **Ordenação por nome ou por idade (com botão activo).** Os dois `<button>` no `<header>` que vinham da Sessão 2 com `console.log` ficam finalmente ligados: declaramos `const [sortBy, setSortBy] = useState("name")`, e cada botão chama `setSortBy("name")` ou `setSortBy("age")`. Derivamos `sorted = [...filtered].sort(...)` no _render_ (o _spread_ é essencial: `.sort()` muta) com `localeCompare` para nomes e subtração para idades, e o `.map()` itera `sorted`. O botão activo destaca-se via _ternary operator_ em `className` (`bg-blue-600 text-white` vs estilo neutro), mesmo padrão da Sessão 4 para o _card_ seleccionado.
